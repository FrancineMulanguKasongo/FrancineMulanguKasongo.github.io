# Reflection on Coding in Markdown

## Situation
As a final-year ICT student at CPUT, I needed to create my CV using GitHub Markdown for my PRP370 assessment. I had never coded a CV in Markdown before.

## Task
My task was to convert my traditional CV into Markdown format, ensuring proper formatting for headings, lists, links, and professional presentation.

## Action
- I learned Markdown syntax by reading GitHub's documentation
- I started with basic headings using # symbols
- I used **bold** and *italic* formatting for emphasis
- I created bullet point lists using - symbols
- I added links using [text](URL) syntax
- I tested my formatting by previewing the file
- I made adjustments to improve readability

## Result
I successfully created a professional-looking CV in Markdown format. I learned that Markdown is simple yet powerful for documentation. This skill will help me in future IT projects for creating README files and technical documentation.
